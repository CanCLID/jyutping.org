## ローマ字表記法を設計する際の原則

広東語は漢字で書かれる言語の一つですが、漢字は表音文字ではないため、その発音を表すためのシステムが必要です。そんなシステムのうち表記にアルファベットを用いるものを「ローマ字表記法」といい、発音を特定の表記法を用いて書き写すことを「転写」といいます。ローマ字表記法の最優先事項は、言語の音韻体系、つまり機能的な音の体系を正確に転写することです。インターネット時代である21世紀において、ローマ字表記法はインプットメソッド（IME）などデジタル化ツールを作成するためのピースという役割も担っています。この記事は、これらの実用的な観点から、ローマ字表記法を設計するための一連の原則と基準をまとめ、一つずつ深く掘り下げます。また、これらの原則に則って既存の広東語ローマ字表記法を検討し、なぜ粤拼が推奨されるべきかを説明します。

### 原則1：「一つの記号は一つの音素に対応」

言語の音韻体系を正確に表すために、ローマ字表記法は**一つの記号は常に一つの音素にのみ対応し、その逆も同様**という規則に厳密に従わなければなりません。一つの記号が複数の音素を表す、または一つの音素が複数の記号で表される場合、そのシステムは一貫性がなく、音韻体系を正確かつ簡潔に表せていないと言えるでしょう。

よく知られた例は普通話ピンインです。普通話ピンインにおいて文字 e は音素転写であり、声母 g k h の後では /ɤ/ と発音され、韻母 ie では /e/ と発音されます。一方、声母 j q x は音声転写であり、g k h と同じ音素を共有しています。つまり、同じ音素が g k h または j q x のいずれかで表されるのです。音の転写を時には音素的に、時には音声的に行っていることから、普通話ピンインは一貫性を欠き、原則1に違反していることがわかります。

原則1は、記号が表す音は**その記号の環境（前後の記号）に依存すべきではない**ということをも意味します。例えば広東語のイェール（耶魯）式ローマ字表記において、文字 a は2つの母音音素 /a/ と /ɐ/ を表すことができます。下の表は、「花」「翻」「分」の3つの漢字のイェール式ローマ字表記を示しています。文字 a は、開音節（子音で終わらない音節）の漢字（表中の「花」）では母音 /a/ を表している一方、閉音節の漢字では別の母音音素 /ɐ/（表中の「分」）を表しています。言い換えると、1つの記号 a が2つの音素 /a/ と /ɐ/ に対応しています。同様に、「花」と「翻」は同じ母音 /a/ を共有していますが、「花」では fa と転写され、「翻」では faan と転写されます。つまり、1つの音素 /a/ が2つの記号 aa と a で表されているのです。

### 原則2：非ASCII文字の使用を避ける

ローマ字表記法は、記号の使用を [ASCII コード](https://ja.wikipedia.org/wiki/ASCII)（American Standard Code for Information Interchange）のみに制限するよう努めるべきです。言い換えれば、26の英字とアラビア数字のみを使用し、´ˋˇ˘˜ のような発音区別符号の使用を避けるべきです。

身近な例といえば、普通話ピンインにおける ü でしょう。ご存知のとおり、この ü を入力するのは非常に面倒で、普段この記号を使う際はネットからコピー＆ペーストしているのではないでしょうか。ü という記号は入力が煩雑なだけでなく、ASCII 文字セットに含まれていないため、（v に変換しない限り）最も単純な形式でエンコードすることができないという問題もあります。このように非 ASCII 文字を使用すると、メモリ使用量とバグが発生するリスクの両方を増加させます。普通話ピンインはまた、声調を示すために ˉˊˇˋ の4つの発音区別符号を使用しますが、入力が不便でミスも起こりやすくなります。とはいえ、普通話ピンインはコンピュータの普及以前である1950年代に設計されたものであり、こうした問題が想定外であっても当然かもしれません。

一方、ローマ字表記法には（日本語入力のように）インプットメソッド（IME）で使われるという周知の用途があります。もし表記法に â ä ô ö û ü といった子音や母音を表す記号が散在していたら、あまりの複雑さにインプットメソッドとして成立しないでしょう。

### 原則3：記号の選択はできるだけ一般的な習慣に従う

発音に対応する文字を選ぶ際は、既存の認識にできるだけ従うべきです。例えば、文字 a o e は母音を表し、b p m f は唇音を表すなどです。

粤拼は、文字 j を使って発音 /j/ <Audio client:load id="jaa1" /> を表しているという点でよく批判されています。多くの人が文字 y で発音 /j/ を表すことに慣れているため、この原則に違反しているというものです。この点については[後ほど](#j-と-y-について)詳しく説明します。

### 原則4：言語の音韻体系を正確に反映する

これは最も重要な設計原則と言えますが、最も厄介で問題が起こりやすいものでもあります。音韻分析にはある程度の主観性があるため、言語の音韻体系を完全に確定することができない場合があるからです。

## 他の広東語ローマ字表記法の欠陥

ローマ字表記法の設計原則を理解したうえで、イェール（耶魯）式、広州話拼音方案（饒秉才方案）、教育学院拼音方案（教院式）という3つのよく知られている広東語ローマ字表記法を見てみましょう。それぞれの設計上の問題を分析し、なぜ粤拼が最適解なのかを説明します。

### イェール（耶魯）式

イェール式は粤拼以外で最も一般的な広東語ローマ字表記法です。粤拼と比較した主な違いは以下のとおりです。

1. イェール式では、/a/ は開音節で a、閉音節で aa と表記されます。/ɐ/ は閉音節で a と表記され、開音節の /ɐ/ は表記できません。
2. イェール式は j ch y で /ts/、/tsʰ/、/j/ をそれぞれ表します。また、母音 /yː/ は yu で表す一方、/j/ と /yː/ を合わせた /jyː/ を yyu ではなく yu と書く特殊ルールがあります。
3. イェール式は /ɵ/ と /œ/ を区別せず、両方とも eu で表記します。そのため、イェール式は /ɛːu/ の音を表記できません。
4. イェール式は付加記号 ˉˊ と文字 h を使って声調を表します（新版では数字による声調表記が追加されました）。

### 広州話拼音方案（饒秉才方案）

広州話拼音方案は1960年に広東省政府が公布した広東語ローマ字表記法で、1980年に改訂版が出されました。声調を除く粤拼との主な違いは以下のとおりです。

1. e ê é ü を使ってそれぞれ母音 /ɐ/、/œː/、/ɛː/、/yː/ を表す
2. y を使って子音 /j/ を表し、gu、ku で /kʷ/、/kʷʰ/ を表す
3. z c s と j q x の両方を使って /ts/、/tsʰ/、/s/ を表す
4. 入声韻尾を -b -d -g で表す

多くの付加記号を使って母音を表していることから、第一点が原則2に完全に違反していることは明らかです。これはコンピュータ情報処理を著しく妨げ、インプットメソッドとして使用することもできません。

### 教育学院拼音方案（教院式）

教育学院拼音方案はイェール式と粤拼以外でよく使われる表記法の一つです。声調を除く粤拼との主な違いは以下のとおりです。

1. dz と ts を使って /ts/ と /tsʰ/ を表す
2. y を使って /y/ を表す
3. /ɵ/ と /œ/ を区別せず、oe を使って表す
4. oey を使って粤拼の eoi を表す

教院式は総じて粤拼と似かよっており、その設計も科学的だといえます。2.と4.はいずれも主観的な記号の選択であって、欠点とは言えません。3.については前述しました。唯一小さな欠点となるのは1.です。一つの声母を表すのに dz や ts と 2文字を使うのは不必要かつ冗長なだけでなく、インプットメソッドのコーディングにも影響します。声母による短縮入力機能は多くのインプットメソッドに搭載されていますが、たとえば ts と入力した場合、「此」という1文字の声母とするか、「聽寫」という2文字の声母とするかが確定できません。コードの重複率が上がるだけでなく、処理コストも上がり、インプットメソッドとしての使い勝手も悪くなります。

## 粤拼の欠陥

### j と y について

粤拼で最も批判されるのは j です。「なぜ /j/ を表すのに y を使わないのか」は最も聞かれる質問といっても過言ではないでしょう。普通話ピンインでも英語でも y で /j/ を表すので、一般的な習慣に従う観点からは、粤拼もそうすべきかもしれません。それでも粤拼はあえて原則3には目をつぶって j で /j/ を表し、y を使わないことにしました。それは26しかない英字のうち、y を /j/ に使ったら、/yː/ を表すのに何を使えばいいのか、という問題が生じるからです。

この問題に対して、イェール式では「y で /j/ を表し、また環境により /yː/ を表す」という答えが導き出されました。文字 y の後に u が続く場合、2文字 yu を組み合わせて /yː/ を表します。つまり、「1つの文字が1つの単母音を表し、2つ組み合わせると別の単母音を表す」という道を選んだイェール式は、**原則3を得るために原則1を犠牲にしている**のです。しかし、ローマ字表記法の目的である「言語の音韻体系を正確に転写する」を達成するには、間違いなく原則1「一つの記号は一つの音素に対応」がより重要ではないでしょうか。

一方、j で /j/ を表すことが原則3に違反するといっても、それはあくまでも限られた文脈での話です。y で /j/ を表すのは普通話ピンインと英語の書記習慣に過ぎないからです。世界には他にも多くの言語の書記システムがあり、たとえばドイツ語では j で /j/ を表します。そもそもこの硬口蓋接近音を表す国際音声記号（IPA）は /j/ であって、粤拼での表記法は国際音声記号の書き方に沿ったものといえるわけです。

### 韻母 yu

粤拼に本当の設計上の欠陥があるとすれば、/yː/ を2文字の yu で表すことでしょう。これは冗長で不必要であり、y の1文字だけで副作用なく解決できます。LSHK が当初このように2文字で表記するよう設計したのは、一般的な習慣と歴史的な背景（原則3）を考慮したためです。幸い現在一般的に利用可能な粤拼インプットメソッドのほとんどが y を yu の省略形としてサポートしているため、入力時に yu の二文字を打つのが手間だと思われる場合は、y だけを打っても全く問題ありません。
