## ローマ字表記システムの設計原則

広東語は漢字（中国語の文字）を書記体系として使用しています。しかし、漢字は表意文字であり、表音文字ではありません。そのため、広東語の音を表すための表音転写システムが必要です。これがローマ字表記システムです。したがって、ローマ字表記システムの最優先事項は、言語の音韻論、つまり音体系を正確に転写することです。さらに、21世紀のインターネット時代において、ローマ字表記システムは入力方法（IME）やその他のデジタル化ツールの作成にも役立ちます。これらの実用的な観点から、ローマ字表記システムを設計するための一連の原則と基準をまとめることができます。これらの原則を詳しく見て、既存の広東語ローマ字表記システムを検討し、なぜ粤拼を選ぶべきかを説明します。

### 原則1：「一つの記号は一つの音素に対応」

言語の音韻論を正確に表すために、ローマ字表記システムは**一つの記号は常に一つの音素にのみ対応し、その逆も同様**という規則に厳密に従うべきです。一つの記号が複数の音素に対応できる場合、または一つの音素が複数の記号で表される場合、そのシステムは一貫性がなく、音韻論を正確かつ簡潔に表すことができないと言えます。

よく知られた例は普通話ピンインです。ピンインでは、文字 e は音素転写です。声母 g、k、h の後では /ɤ/ と発音され、韻母 ie では /e/ と発音されます。しかし、ピンインでは声母 j q x は音声転写です。これらは g k h と同じ音素を共有しており、同じ音素が g k h または j q x のいずれかで表されることを意味します。ピンインは音を音素的にも音声的にも転写しています。これは一貫性を欠き、第一原則に違反しています。

原則1はもう一つの経験則も示唆しています：記号が表す音は**その記号の環境（前後の記号）に依存すべきではない**ということです。良い例は広東語の耶魯（イェール）式ローマ字表記です。イェール式では、文字 _a_ は二つの母音音素 /a/ と /ɐ/ を表すことができます。下の表は、花、翻、分の三つの漢字のイェール式ローマ字表記を示しています。開音節（子音で終わらない音節）の漢字、この場合は「花」では、文字 a は母音 /a/ を表します。一方、閉音節の漢字では、別の母音音素 /ɐ/（表中の「分」）を表します。これが「一つの記号 _a_ が二つの音素 /a/ と /ɐ/ に対応する」というものです。同様に、「花」と「翻」は同じ母音 /a/ を共有していますが、「花」では _fa_ と転写され、「翻」では _faan_ と転写されます。これは「一つの音素 /a/ が二つの記号 _aa_ と _a_ で表される」ということです。

したがって、イェール式ローマ字表記は「記号が実際に表す音を環境に依存して決定する」完璧な例です。文字 _a_ が /a/ を意味するか /ɐ/ を意味するかは、音節が開音節か閉音節かによって決まります。これは多くの曖昧さと不一致を生み出し、イェール式ローマ字表記システムの重大な欠陥の一つでもあります。粤拼では、/a/ は常に **aa** と転写され、**a** は環境に関係なく常に /ɐ/ を表します。

### 原則2：非ASCII文字の使用を避ける

ローマ字表記システムは、記号の使用を [ASCII コード](https://ja.wikipedia.org/wiki/ASCII)（American Standard Code for Information Interchange）のみに制限するよう努めるべきです。言い換えれば、システムは26の英字とアラビア数字のみを使用し、´ˋˇ˘˜ のような発音区別符号の使用を避けるべきです。

完璧な例は普通話ピンインの ü です。アメリカ式キーボードで文字 _ü_ を入力するのがどれほど面倒かは誰もが知っています。ほとんどの人は特殊文字入力アプリを探して入力する必要があります。面倒なだけでなく、_ü_ は ASCII コードセットに属さないため、最も単純な形式でエンコードできません（_v_ に変換しない限り）。これはメモリ使用量とバグのリスクの両方を増加させます。ピンインはまた、声調を示すために ˉˊˇˋ のような発音区別符号を使用しますが、これも _ü_ と同様に ASCII エンコーディングでサポートされていません。

### 原則3：記号の選択はできるだけ一般的な習慣に従う

この原則の意味は、発音に対応する文字を選択する際、既存の認識にできるだけ従うべきだということです。例えば、文字 a o e は通常母音を表すために使用され、b p m f は唇音を表すなどです。

粤拼がよく批判されるのは、文字 j を使って発音 /j/ <Audio client:load id="jaa1" /> を表すことです。これは、皆が文字 y で発音 /j/ を表すことに慣れているため、この原則に違反しています。この点については[下記](#j-と-y-について)で詳しく説明します。

### 原則4：言語の音位体系を正確に反映する

これは最も重要な設計原則と言えますが、最も厄介で問題が起こりやすいものでもあります。音位分析にはある程度の主観性があり、言語の音位体系を完全に確定することができない場合があります。

## 他の広東語ローマ字表記方案の欠陥

ローマ字表記方案の設計原則を理解した後、最も有名な広東語ローマ字表記方案をいくつか見てみましょう：耶魯（イェール）式、教育学院式、広州話拼音方案（饒秉才方案）。それぞれの設計上の問題を分析し、なぜ粤拼を選ぶべきかを証明します。

### 耶魯（イェール）式

イェール式は粤拼以外で最も一般的な広東語ローマ字表記方案です。粤拼と比較した主な違いは以下の通りです：

1. イェール式では、/a/ は開音節で a、閉音節で aa と表記されます。/ɐ/ は閉音節で a と表記され、開音節の /ɐ/ は表記できません。
2. イェール式は j ch y を使って /ts/、/tsʰ/、/j/ の三つの音を表します。そのため、母音 /yː/ は yu で表す必要があります。
3. イェール式は /ɵ/ と /œ/ を区別せず、両方とも eu で表記します。そのため、イェール式は /ɛːu/ の音を表記できません。
4. イェール式は付加記号 ˉˊ と文字 h を使って声調を表します（新版では数字による声調表記が追加されました）。

### 広州話拼音方案（饒秉才方案）

広州話拼音方案は1960年に広東省政府が公布した広東語ローマ字表記方案で、1980年に改訂版が出されました。声調以外で粤拼との主な違いは以下の通りです：

1. e ê é ü を使ってそれぞれ母音 /ɐ/、/œː/、/ɛː/、/yː/ を表す
2. y を使って子音 /j/ を表し、gu、ku で /kʷ/、/kʷʰ/ を表す
3. z c s と j q x の両方を使って /ts/、/tsʰ/、/s/ を表す
4. 入声韻尾を -b -d -g で表す

明らかに、第一点は原則2に完全に違反しており、多くの付加記号を使って母音を表しています。これはコンピュータ情報処理を著しく妨げ、入力方法として使用することもできません。

## 粤拼の欠陥

### j と y について

粤拼で最もよく批判されるのは j です。多くの人がなぜ /j/ を表すのに y を使わないのかと尋ねます。普通話ピンインでも英語でも y で /j/ を表すので、これは確かに一般的な習慣に従っています。では、なぜ粤拼は原則3に違反して j で /j/ を表し、y を使わないのでしょうか？それは、y を使って /j/ を表すと、問題が生じるからです：26の英字しかないのに、y を /j/ に使ったら、/y/ を表すのに何を使えばいいのでしょうか？

この問題に対して、イェール式の答えは：y で /j/ を表し、環境判断で /y/ を表す時を決める、というものです。文字 y の後に u が続く場合、二つの文字 yu を組み合わせて /y/ を表します。これが上述の「一つの文字が一つの単母音を表し、二つ組み合わせると別の単母音を表す」というものです。つまり、**イェール式は原則3を得るために原則1を犠牲にしている**のです。では、どちらの原則がより重要でしょうか？間違いなく第一条「一つの記号は一つの音素に対応」が最も重要です。

一方、j で /j/ を表すことはそれほど原則3に違反する選択ではありません。y で /j/ を表すのは普通話ピンインと英語の書記習慣に過ぎないからです。世界には他にも多くの言語の書記システムがあり、例えばドイツ語も j で /j/ を表します。さらに、国際音声記号ではこの音（硬口蓋接近音）の記号は /j/ です。したがって、粤拼がこのように書くのは国際音声記号の書き方に従っているのです。

### 韻母 yu

粤拼に本当の設計上の欠陥があるとすれば、それは二つの文字 yu を使って母音 /y/ を表すことです。これは冗長で不必要に見え、一つの y だけで副作用なく解決できます。LSHK が当初このように二つの文字を使用するよう設計したのは、一般的な習慣と歴史的な問題（原則3）を考慮したためです。しかし、良いニュースは、現在市場に出回っている粤拼入力方法のほとんどが y = yu の省略形をサポートしているため、入力時に yu の二文字を打つのが面倒な場合は、y だけを打っても全く問題ありません。
